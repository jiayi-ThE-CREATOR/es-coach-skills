# shukatsu-distill — 就活知識蒸留・素材化ツール

日本就活の内定者経験談を蒸留し、ES添削・模擬面接を行うスキル。

## パス解決（正本はレジストリ）

パスの**唯一の定義元**は `$SHUKATSU_SKILLS_ROOT/vault.paths.env`（このリポジトリをクローンした場所を指す環境変数。README のセットアップ手順を参照。vault 再編成時はそこだけ更新される）。対応キー：

```
VAULT_SHUKATSU_DISTILL    → 就活蒸留システムの根目録（素材/蒸留知識/添削ログの共通の親）
VAULT_SHUKATSU_SOZAI_YT   → 素材/YouTube/
VAULT_SHUKATSU_SOZAI_WEB  → 素材/就活サイト/
VAULT_SHUKATSU_SOZAI_SELF → 素材/本人/（grill モードの保存先）
VAULT_SHUKATSU_CHISHIKI   → 蒸留知識/内定之路/（成功パターン .md はこの配下）
VAULT_SHUKATSU_HIKAKU_LOG → 添削ログ/
```

**以降の手順に出てくるパスが存在しなかったら、ここを推測で直すのではなくレジストリを読んで解決し直すこと。**

## モード選択

引数を確認する：
- `ingest` → ingestモードへ
- `distill` → distillモードへ
- `coach mock` → 模擬面接モードへ
- `grill` → grillモードへ（逆面接で本人の頭の中から素材を抽出する）
- 引数なし → ユーザーにどのモードか確認する
- ES添削は `/es-coach` スキルを案内する

---

## ingest モード

**目的**: 経験談・動画文字起こし・就活サイト記事を素材として保存する。

### 手順

**Step 1 — 素材の受け取り**

ユーザーが提供するものを確認する：
- テキスト貼り付け → そのまま使用
- URL（YouTube / 就活サイト / Unistyle / 外資就活 / Voluntas等）→ defuddleスキルでMarkdown抽出を試みる。失敗した場合はWebFetchで直接取得する

**Step 1.5 — 重複チェック**（URLが提供された場合のみ）

以下のコマンドで既存ファイルに同じURLが存在しないか確認する：

```bash
source "$SHUKATSU_SKILLS_ROOT/vault.paths.env"
grep -rl "source: [入力されたURL]" "$VAULT_SHUKATSU_DISTILL/素材/"
```

既存ファイルが見つかった場合：「このURLはすでに `[ファイル名]` として保存されています。再ingestionしますか？」と確認してから続ける。

---

**Step 2 — メタデータ抽出**

素材から以下を読み取る（不明な場合は「不明」と記載）：

```
- 内定先: [企業名・業界]
- 業界分類: [金融 / 商社 / コンサル / IT / 全行業共通 / 複合]
- ソース種別: [YouTube / 就活サイト / その他]
- ソースURL: [あれば]
- 内定数: [何社か記載があれば]
- 就活時期: [何年度か]
```

**Step 3 — 業界分類の判定**

素材の主な対象を判定する：
- 外資投資銀行・証券・メガバンク・信託・保険 → `金融`
- 総合商社・専門商社 → `商社`
- 戦略コンサル・総合コンサル・会計系アドバイザリー（McK / BCG / PwC / デロイト等）→ `コンサル`
- IT（ハードウェア / ソフトウェア / インターネット・Web / 通信インフラ / 情報処理サービス・SIer）→ `IT`
- 複数業界にまたがる / 就活全般のコツ → `全行業共通`
- 複数業界にまたがる → `複合`（該当業界すべてにタグ付け。例：金融×IT の証券システム／SE 系）

**Step 4 — ファイル保存**

保存先を決定する：
- YouTube動画の文字起こし → `素材/YouTube/`
- 就活サイト記事・ブログ → `素材/就活サイト/`

ファイル名: `YYYY-MM-DD_[内定先略称]_[業界].md`
例: `2026-05-08_三菱商事_商社.md`

ファイル内容フォーマット：
```markdown
---
title: [タイトル or 動画タイトル]
source: [URL or "貼り付け"]
source_type: [YouTube / 就活サイト / その他]
industry: [金融 / 商社 / コンサル / IT / 全行業共通 / 複合]
company: [内定先]
offer_count: [内定数 or 不明]
year: [就活時期 or 不明]
added: YYYY-MM-DD
distilled: false
---

# [タイトル]

## 原文・要約

[素材の全文 or 要約（できるだけ原文を保持）]

## 抽出メモ（ingest時点の速報）

[明らかに重要だと思った点を3〜5点箇条書き]
```

**Step 5 — 報告**

以下を報告する：
- 保存パス
- 業界分類
- 速報メモ（3点）

**Step 6 — 自動蒸留＋状態報告（必須・ユーザーの恒久指示）**

ユーザーから「素材を追加したら毎回 蒸留知識を同期し、更新の有無とフォルダが最新状態かを報告せよ」という恒久指示がある。
したがって素材保存後、**確認を取らずに続けて distill モードを実行**する（複数素材をまとめて ingest した場合は全部保存し終えてから1回 distill する）。
distill 完了後、必ず次の2点を明示する：
- **今回の更新**：あり（更新した知識ファイル名＋追記パターン数）／ なし（新規の未蒸留素材なし）
- **蒸留知識フォルダの状態**：✅ 最新データに基づく（未蒸留 0 件）／ ⚠️ 未蒸留 N 件残（ファイル名）

---

## distill モード

**目的**: 素材フォルダの全ファイルを分析し、成功パターンを蒸留知識ファイルに反映する。

### 手順

**Step 1 — 未蒸留素材の確認**

以下のコマンドで `distilled: false` のファイルを先に特定する：

```bash
source "$SHUKATSU_SKILLS_ROOT/vault.paths.env"
grep -rl "distilled: false" "$VAULT_SHUKATSU_DISTILL/素材/" 2>/dev/null
```

素材ルート全体を探索する（`就活サイト/` `YouTube/` `一次情報/` `本人/` `ES_長所短所/` `其他/` すべて対象）。
このコマンドの結果リストのファイルのみを読み込む（全ファイルを読まない）。
未蒸留素材が0件なら「新しい素材がありません」と報告して終了。

**Step 2 — 既存の蒸留知識を読む**

更新対象ファイルを読み込む（実体は `VAULT_SHUKATSU_CHISHIKI` 配下）：
- `全行業共通.md`
- `金融.md`（金融素材があれば）
- `商社.md`（商社素材があれば）
- `コンサル.md`（コンサル素材があれば）
- `IT.md`（IT素材があれば）

**Step 3 — パターン抽出（核心）**

未蒸留素材を読み込み、以下の観点で分析する：

**全行業共通で見るべき観点：**
- 第一印象・声・態度に関する言及
- 自己PR・ガクチカの構成パターン（STAR法など）
- 志望動機の組み立て方
- 面接での話し方・テンポ・論理展開
- 「落ちた人と何が違った」という自己分析
- 準備の量・質（模擬面接回数、OB訪問数など）
- 面接官が「刺さった」と言った瞬間

**金融特有で見るべき観点：**
- 金融を志望した動機の説得力
- 市場・財務知識の水準
- 外資と国内の違いへの理解
- 「お金を扱う」職業への適性をどう示したか
- 面接で問われた数字・ケース問題

**商社特有で見るべき観点：**
- ガクチカのスケール感と選び方
- 主体性・リーダーシップのエピソード型
- 海外・異文化経験の使い方
- 「なぜ商社か」「なぜこの商社か」の差別化
- GD・ケース面接での振る舞い

**コンサル特有で見るべき観点：**
- 「なぜコンサルか」「なぜ戦略/総合/会計系か」の差別化と論理性
- ケース面接・フェルミ推定への準備方法と実際の出題
- 構造化思考・仮説思考をエピソードでどう示したか
- 激務・Up or Out への覚悟をどう伝えたか
- 他業界（投行・商社）併願をどう説明したか

**IT特有で見るべき観点：**
- 5領域（ハード / ソフト / Web / 通信インフラ / SIer）のどこを志望し、なぜその領域かの言語化
- 「なぜITか」「なぜSIerか（自社開発でなく受託か、その逆か）」の差別化
- 技術への興味と、文系でも通用する論理性・課題解決の示し方
- 個社の稼ぎ頭がどの領域か（売上構成）を踏まえた企業研究の深さ
- 金融×IT（証券・銀行システム、勘定系・トレーディング）志望なら、SIerと金融機関システム部門の違いの理解

**Step 4 — 蒸留知識の更新**

各知識ファイルの該当セクションに、抽出したパターンを追記する。

書き方のルール：
- **上書きしない**、既存の内容に追記する
- パターンは「〜という傾向がある」「〜と言っている人が多い」という形で書く
- 素材の出典を`[出典: ファイル名]`の形で括弧内に入れる
- 矛盾するパターンは削除せず「一方で〜という見方もある」と併記する
- 数が増えたら冒頭のsource_countを更新する

更新後、素材ファイルの `distilled: true` に変更する。

**Step 5 — 蒸留サマリーの報告**

以下を報告する：
- 処理した素材数
- 各知識ファイルに追加されたパターンの件数
- 今回の蒸留で見つかった最も重要なインサイト（上位3点）
- 知識ファイルへのパス
- **状態確認（必須）**：Step 1 の grep を再実行して `distilled: false` が 0 件かを確認し、
  「✅ 蒸留知識フォルダは最新データに基づく（未蒸留 0 件）」/「⚠️ 未蒸留 N 件残（ファイル名）」を明示する。
  あわせて「今回の更新：あり／なし」も1行で述べる。

## coach モード

引数を確認する：
- `coach mock` → 模擬面接サブモード
- `coach es` または ES添削の要求 → `/es-coach` スキルを使うよう案内して終了
- 引数なし → 「模擬面接は `coach mock`、ES添削は `/es-coach` をお使いください」と案内

---

### coach mock — 模擬面接

**目的**: 実際の面接を模擬し、蒸留知識に基づいて回答をフィードバックする。

**Step 1 — セッション設定**

以下を確認する：
- 志望企業・業界（金融/商社/その他）
- 面接種別（1次面接 / 最終面接 / インターン選考）
- 時間（何問やるか、または「終わりにしたいとき言って」）

**Step 2 — 蒸留知識の読み込み**

業界に応じた知識ファイルを読む（ES添削と同様）。
頻出質問リストを内部で構築する。

**Step 3 — 面接進行**

以下のルールで進行する：
- 1問ずつ質問する（面接官口調：「〜について教えてください」）
- ユーザーが回答したら、即時フィードバックを出す
- フィードバックフォーマット：
  ```
  [評価: 良い / もう少し / 要改善]
  良かった点: ...
  改善点: ...（蒸留知識と照合した具体的指摘）
  内定者はここでこうする: ...
  ```
- ユーザーが「次」と言ったら次の質問へ
- ユーザーが「終わり」「ここまで」と言ったら総評へ

質問の順番（業界に関係なく共通）：
1. 自己紹介（1分）
2. ガクチカ
3. 自己PR
4. 志望動機（業界）
5. 志望動機（企業）
6. 逆質問

業界特化質問（金融）：
- 「なぜ金融か、なぜ銀行か証券かを教えてください」
- 「最近気になるマーケットニュースはありますか」
- 「5年後・10年後のキャリアイメージを教えてください」

業界特化質問（商社）：
- 「学生時代の最大の挫折と乗り越え方を教えてください」
- 「なぜ商社か、なぜ弊社かを教えてください」
- 「海外でどのような仕事がしたいですか」

業界特化質問（コンサル）：
- 「なぜコンサルか、なぜ弊社かを教えてください」
- 「（ケース）日本のコンビニコーヒーの市場規模を推定してください」
- 「クライアントの意見とデータが食い違うとき、どう動きますか」

業界特化質問（IT）：
- 「なぜIT業界か、5領域のうちどこを志望するかを教えてください」
- 「弊社の事業のうち、どの領域に関わりたいですか。なぜですか」
- 「文系のあなたが、技術者と協働するうえで発揮できる強みは何ですか」

**Step 4 — 総評**

セッション終了時に出力する：

```
## 模擬面接 総評

### 強み（このまま使える）
- ...

### 要強化ポイント（優先順）
1. ...
2. ...

### 内定者との最大の差分
[1つだけ、最も重要な差を指摘]

### 次のセッションまでにやること
- ...
```

## grill モード — 逆面接（知識抽出）

**目的**: ユーザーの頭の中にしかない経験・価値観・エピソードを、AI からの集中質問（15〜30問）で掘り出し、蒸留システムの素材として保存する。内定者パターン（蒸留知識）は「他人の成功」、grill は「本人の原石」——両方揃って初めて ES・面接の個別戦略が組める。

### 手順

**Step 1 — テーマ設定**

ユーザーに一つ選ばせる（または指定を受ける）：
- `自己分析` — 価値観・強み弱み・人生の転機
- `ガクチカ深掘り` — 1つのエピソードを面接官視点で掘り尽くす
- `志望動機` — Why 金融/商社・Why この会社・Why あなた の接続を固める
- `弱点つぶし` — 想定問答で答えに詰まる質問を先に洗い出す

**Step 2 — 蒸留知識の事前読み込み**

`VAULT_SHUKATSU_CHISHIKI` 配下の該当業界ファイルを読み、内定者パターンとのギャップを突く質問を設計する（例：内定者は挫折経験を定量化して語る傾向→「その失敗、数字で言うとどれくらいの損失？」）。

**Step 3 — 逆面接の進行**

- **1問ずつ**。回答を受けてから次を出す。安易に納得しない：抽象的な回答には「具体的には？」「固有名詞で言うと？」「そのとき何を感じた？」「なぜそうした？」と最低1段は掘る
- 良い素材（数字・固有名詞・感情の動き・因果・意外性）が出たら「📌 素材として記録」と宣言して控える
- 15問を超えたあたりで「続けるか、ここまでの分を保存するか」を確認する
- ユーザーが「終わり」「ここまで」と言ったら Step 4 へ

**Step 4 — 素材ファイル保存**

保存先：`VAULT_SHUKATSU_SOZAI_SELF`（素材/本人/）
ファイル名：`YYYY-MM-DD_grill_[テーマ].md`

```markdown
---
title: grill セッション — [テーマ]
source: grill（逆面接）
source_type: 本人
industry: [金融 / 商社 / コンサル / IT / 全行業共通]
theme: [テーマ]
added: YYYY-MM-DD
distilled: false
---

# grill — [テーマ]（YYYY-MM-DD）

## 抽出された素材（Q&A 形式、本人の言葉をできるだけ原文保持）

### Q1: [質問]
[回答の要点＋本人の表現]

…

## セッション所見

- ES・面接にそのまま使える発見: …
- 内定者パターンとのギャップ: …
- まだ掘れていない領域（次回の grill 候補）: …
```

**Step 5 — 自動蒸留＋状態報告（ingest Step 6 と同じ恒久指示）**

保存後、確認を取らずに distill モードを実行し、「今回の更新あり/なし＋蒸留知識フォルダの状態」を報告する。本人素材は `[出典: grill_ファイル名]` として蒸留知識に追記される——内定者パターンと本人の実体験が同じ知識ベースで突き合わせ可能になる。
